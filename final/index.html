Exploratory Project


My exploratory project started off using data on cetaceans in captivity across the U.S. After having some difficulty figuring out how to present this in different charts that synced, I decided to use the same data I had visualized for the time series tutorial – hate groups, as tracked by the Southern Poverty Law Center (SPLC), an advocacy organization based in Alabama. By the time of the exploratory project, they had updated their database to include active hate groups in 2019. At the start of the project, I planned to include a per capita map, like the "hate map" found on their site, which when clicked by state brings up a list of specific hate groups in the state. I wanted to recreate this in D3, in a more basic format limited to what I could produce, while also add a visualization that would aggregate the number of groups in each state by ideology. I considered various combinations of visuals:

Eventually, I settled on a bar chart for the aggregated data, in addition to the map and table. When a state is clicked on, the two other charts would show the number of groups by belief and a listing of all the groups, respectively. This dashboard of sorts would target anyone with an interest in hate groups at a state level especially amid a political and social environment of heightened organized hatred. The data provided the names of all groups tracked by the SPLC along with their location, ideology and other details. In order to get the data suitable for the map, I used R to manipulate it. Using the `tidycensus` package, I got each state's population and then divided the total number of groups by that number to find the the per capita number of hate groups for each state. This produced the hg.csv file, which would be used for the map and the table. While the number of groups per ideology in a state could be aggregated from that, I created another file instead in R as my attempts to reconfigure the data in D3 did not work. This second file, hg_ideology_across.csv, aggregated the number of groups by ideology for each state and was used for the bar chart. I also used R to make a quick dashboard of how the visuals would be laid out.

Heading into the coding process seemed rather intimidating but drawing the skeletal diagram below helped organize it a bit. At that point I was still toying with the idea of a possible treemap but that quickly disappeared once I hit the actual coding.

Getting the choropleth map to appear initially seemed challenging enough but it would turn out to be relatively smooth compared the great difficulty I had getting the three visuals to sync. Once the map appeared with the color scale I intended and each state was being properly read as the actual state, I moved on to the bar chart. I encountered the following issues:

  • Getting the bars to fit in the condensed space allotted for the chart
  • Having the ideology numbers and names appear to the right of the bars
  • Getting the information to change based on the state clicked
  • Getting the numbers to start out as 0 and then change to the right number when a state was clicked on

The table started out less challenging to put on the page but then the roadblock came in getting the hate group list of the original state to disappear when a new state was clicked on. Feeling the need to step away from that after a while, I focused on the design. I wanted to use red or blue colors to signify that these are U.S. hate groups and settled on a warm red-orange-beige color palette as red has a connotation of alarm and distress but I wanted a more subtle palette than one dominated by a bold red.

For the bar chart I added a color range to mirror the map but not replicate the colors exactly. I chose the font primarily to fit all the text in the tight spaces for the visuals. This was the same reasoning for choosing only three fields for the table. Meanwhile, I thought of different orientations for the three charts including putting the map in the middle and slightly above the other two but by then in the project, I just wanted everything to sync properly. Eventually, many weeks later, they did as I had, more or less, intended. At a certain point, it came down to playing around with the code and seeing what worked. Then, I could add some extra elements that would actually tell the user the state they accessed – a tooltip and a button in which the state name would appear until another one was clicked on.

The design process would pretty much mirror the one taught in class from writing and sketching it out whether on paper or using computer software. Since learning about this process, I've paid more attention to videos and podcast episodes and blog posts on the benefits of sketching out ideas. Going forward I'll add these drawings and notes to my process. I believe this, along with the code that produced the end product, became the most helpful elements I learned. And of course, every step served as vital practice particularly in trial and error. The final dashboard more or less stayed in line with what I had in mind during the prospectus stage.

To improve my exploratory project, I would try to experiment with other layouts, eliminate the NULL and black square of the tooltip that appears when the page first loads, and having some kind of message in the bar and table charts that would explain the information that would appear once a state is clicked on. I would also figure out how to add more details to the tooltip for the map and format the table content differently.

Narrative Project


For my narrative project, I used to same data source as above, the Southern Poverty Law Center, but focused on New York State hate groups. Where are these groups located and how does NYC compare to the rest of the state? What are the most frequent ideologies followed and how do they compare to the country as a whole? Have the most active ideologies changed over the past decade? I started out, like above, intent on having a map as one of the visualizations. The rest would include some combination of the standard charts.

The audience would be the same as my exploratory project. And again, I did a bit of data manipulation in R. I ended up with the following datasets, some of which were created later on in the process after playing around with the charts.

  • hg_nys_2019.csv: list of all 44 NYS hate groups in 2019, with their ideology, location and whether it served as the headquarters for a table
  • hg_ideology_pct_2019.csv: Percent of each ideology out of total hate groups in NYS and US for a grouped bar chart
  • hg_nys_geocoded_2019.csv: List of cities with at least one hate group, their geographical coordinates, along with population and number of each ideology for the map and tooltips (with some HTML included in some of the fields for some formatting on the tooltip)
  • hg_nys_ideologies.csv: List of all ideologies and their number of groups in NYS since 2000 for a line chart.

This step helped me to narrow down the visualizations I would use, shown in the mockup.

This time the charts were much easier to make since they stood alone. I had not used CSS for many years before the course but thanks to a few months of practice, I was able to spend more time on it in this project - especially to lay out the charts and text, and find a combination of serif and sans-serif fonts. Different from the other assignments and projects, this was a more word-heavy report so I enjoyed this additional task of focusing on how to present the text. I also took more time to apply formatting to the visualizations that would be consistent to communicate that they are all part of the same project. I considered making more comparisons between NYS and the US, as I did in the line chart, but that could instead be something for improving and building on the project in the future. For this assignment, in addition to putting the visualizations together, an especially helpful element I learned was the CSS grid layout and how to put D3 charts and text side-by-side.

The final product turned out to follow the prospectus. I did have initial concerns about how 44 data points would translate into the visualizations but that size helped shaped my narrative and a change from working with larger datasets. The critiques helped inform me of other much-needed perspectives and user experiences, and served as a valuable reminder that even if I see it one way, no one else necessarily will. After working on a project for a while, you reach a point where you need new eyes.

To improve my narrative project, I would:

  • add more user interaction features such as sorting the bars and the table contents
  • add chart annotations using d3-annotation or something similar
  • make the map have a solid black outline with a lighter interior color

For future projects, I will model my design process after the one taught in class as I realized its importance through these assignments, especially to go back and refer to regardless of what stage you're in. D3 will always be an ongoing learning experience and I look forward to looking back on the class projects and having a better understanding of how exactly everything worked.
